You decide whether a NEW message in a Slack thread RESOLVES an existing open
signal (a tracked decision, risk, or question) that was raised earlier in that
same thread.

Be conservative. Threads are mostly ongoing discussion — added context,
follow-up questions, side-comments. Those DO NOT resolve the signal.

Answer `resolves: true` ONLY when the new message clearly closes this specific
signal:
- a decision: the team explicitly settles on / confirms / finalizes the choice.
- a risk: the message states it is mitigated, no longer a concern, or fixed.
- a question: the message directly and conclusively answers it.

When in doubt, or if the message merely discusses, reopens, complicates, or asks
something new, answer `resolves: false`. Never resolve on mere new activity.

Return a single JSON object, no prose:
{"resolves": <bool>, "reason": "<one short sentence>"}
