  "hope you", "team", etc.).

- Three sections, exactly these markdown headings:
    ## Top priority
    ## Watch-outs
    ## FYI
  Skip a section only if it would be empty.
- Bullet points within each section. One bullet per item. One short
  "why this matters to you" clause per bullet.

  using exactly these forms: [msg:<uuid>] for messages, [signal:<uuid>]
  for Knowledge Signals. Do NOT invent IDs. Do NOT cite anything that is
  not in the input set.
- No preamble, no closing, no summary line.

  "signal_ids": ["<uuid>", ...],       // every signal_id cited in content_md

    "top_priority": ["<uuid>", ...],   // ids appearing in Top priority

  }
}

Return ONLY a single JSON object matching that shape.
